QuickRTF Editor

超轻量、安全的 RTF 文档编辑工具

支持富文本编辑、加密存储（SRTF 格式）、Markdown 导出、快速存档，满足个人文档编辑与安全存储需求

核心功能

富文本编辑

提供完整的 RTF 编辑功能，支持文本格式自定义，满足日常文档编辑需求：

  • ✅ 字体样式调整（粗体、斜体、下划线、删除线）
  • ✅ 字体大小自定义（直接输入或增减调整）
  • ✅ 字体选择（支持系统已安装的所有字体）
  • ✅ Tab 键支持（自动插入空格，优化排版）
  • ✅ 支持 RTF 格式原生保存与打开

SRTF 加密存储

独创 SRTF（Secure RTF）格式，通过 AES 加密保护敏感文档，防止未授权访问：

  • 🔒 采用 AES-256 加密算法，基于用户密钥生成哈希密钥，加密强度高
  • 🔑 支持自定义密钥（留空则使用默认密钥），密钥丢失将无法解密
  • 📁 加密后的 .srtf 文件可安全存储或传输，避免文档内容泄露
  • 🔓 支持直接打开 .srtf 文件，输入正确密钥即可解密编辑

注意：请妥善保管您的加密密钥，密钥丢失后无法恢复文档内容！

快速存档与加载

支持一键快速存档（QuickSave）和加载（QuickLoad），临时文档无需手动保存：

  • ⚡ 快捷键操作：Alt+S 快速存档，Alt+L 快速加载
  • 💾 存档内容以 Base64 编码存储在系统注册表，关闭软件后不丢失
  • 📝 空白文档自动清空存档，避免无效数据占用空间
  • 🔄 支持覆盖存档，随时更新临时文档内容

格式转换与导出

支持多种格式转换，满足不同场景下的文档使用需求：

  • 📤 导出为 Markdown 格式（.md），兼容主流笔记工具（如 Typora、Obsidian）
  • 📋 复制纯 RTF 源码，便于二次开发或格式调试
  • 🖨️ 支持打印功能（注：部分复杂 RTF 格式可能无法完全兼容）
  • 🔗 支持通过命令行打开文件（直接拖放文件到程序图标即可快速打开）

提示：Markdown 导出和打印功能对复杂格式（如表格、图片）支持有限，建议用于纯文本+基础格式文档。

便捷操作体验

优化多项操作细节，提升编辑效率：

  • ⌨️ 全面快捷键支持（Ctrl+S 保存、Ctrl+O 打开、Ctrl+P 打印等）
  • 📌 置顶功能（Toggle TopMost），编辑时不被其他窗口遮挡
  • 🖥️ 响应式设计，支持高 DPI 屏幕，界面显示清晰
  • ❓ 内置帮助文档（EULA、隐私政策、SRTF 格式说明），随时查阅

本工具为绿色软件，无需安装，解压即可使用：

  1. 下载压缩包并解压到任意目录（如 D:\Tools\QuickRTF）
  2. 双击运行 QuickRTF.exe

使用指南与常见问题

基础操作教程

1. 文档编辑

在编辑区直接输入文本，使用工具栏按钮调整格式：

  • 粗体/斜体/下划线/删除线：点击对应按钮（或使用系统默认快捷键 Ctrl+B/I/U）
  • 字体调整：点击「字体」按钮选择字体，通过数字框或「+/-」按钮调整大小
  • Tab 缩进：直接按 Tab 键，自动插入 3 个空格（可在代码中修改）

2. 文档保存与打开

  • 保存为 RTF：点击「文件」→「保存」（Ctrl+S），选择保存路径即可
  • 保存为 SRTF（加密）：点击「文件」→「Save as SRTF」，输入密钥后保存
  • 打开 RTF/SRTF：点击「文件」→「打开」/「Open SRTF」，选择文件后确认（SRTF 需输入密钥）

3. 快速存档与加载

  • 快速存档：编辑完成后按 Alt+S，内容自动存储（无需选择路径）
  • 快速加载：需要恢复存档时按 Alt+L，存档内容会覆盖当前编辑区

4. 格式转换与导出

  • 导出为 Markdown：点击「文件」→「Export to Markdown」，选择保存路径即可
  • 复制 RTF 源码：点击「文件」→「Copy Plain RTF」，源码自动复制到剪贴板

常见问题（FAQ）

Q1：SRTF 文件的密钥忘记了，如何恢复文档？

A1：抱歉，SRTF 采用 AES-256 加密，密钥是解密的唯一凭证，丢失后无法恢复文档内容。建议您为重要文档设置常用密钥并妥善记录。

Q2：导出的 Markdown 格式错乱怎么办？

A2：由于 RTF 格式的复杂性，部分高级格式（如复杂表格、图片、特殊符号）无法完全转换为 Markdown。建议导出前简化文档格式（仅保留文本、基础样式和列表）。

Q3：快速存档的内容保存在哪里？是否安全？

A3：快速存档内容存储在 Windows 注册表的「HKEY_CURRENT_USER\Software\I Am System32\QuickRTF」路径下，仅当前用户可访问，且以 Base64 编码存储，相对安全。但敏感文档仍建议使用 SRTF 加密存储。

Q4：支持打开其他编辑器创建的 RTF 文件吗？

A4：支持！本工具兼容标准 RTF 格式，可打开 Word、WPS、记事本等工具创建的 RTF 文件（复杂格式可能存在轻微兼容问题）。

Q5：如何联系技术支持或反馈问题？

A5：若遇到功能异常、bug 或有功能建议，请发送邮件至 IAmSystem32@outlook.com，注明「QuickRTF+问题描述+系统版本」，我们会在 3 个工作日内回复。

SRTF 格式详解

什么是 SRTF 格式？

SRTF（Secure RTF）是 QuickRTF 专属的加密文档格式，基于标准 RTF 格式扩展，通过 AES 加密算法保护文档内容，适用于存储敏感信息（如密码、隐私笔记、商业文档等）。

SRTF 格式的核心优势：

  • 🔐 加密强度高：采用 AES-256 加密，符合国际安全标准，难以破解
  • 📎 兼容性强：基于标准 RTF 格式，解密后可还原为完整 RTF 内容
  • 🔑 密钥自定义：支持用户设置专属密钥，安全性由用户自主控制
  • 💻 跨平台兼容：加密后的 .srtf 文件可在任何 Windows 设备上通过 QuickRTF 解密打开

加密原理

SRTF 格式的加密流程如下：

  1. 将 RTF 文档内容转换为 UTF-8 编码的字符串，再通过 Base64 编码
  2. 用户输入密钥（或使用默认密钥），通过 SHA-256 哈希算法生成 256 位加密密钥
  3. 使用 AES 算法（CBC 模式 + PKCS7 填充）对 Base64 编码后的内容进行加密
  4. 将加密后的字节流写入文件，后缀名为 .srtf

解密流程与加密相反，需输入相同的密钥才能还原文档内容。
